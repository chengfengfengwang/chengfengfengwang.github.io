又学git

git init 初始化当前目录为仓库，这会生成一个.git文件夹（隐藏文件夹，git运行的文件夹）
git status 获取当前项目状态
git add -A . 或者 git add filename（暂存项目的全部改动，暂存某个文件）将改动放入暂存区
git commit -m 'message'将暂存区内容放入仓库 repository
git remote add origin https://github.com/try-git/try_git.git为当前项目添加远程仓库
git push -u origin master 将本地仓库推送到远程仓库（仓库名为origin 分支为master）-u 会保存当前推送的参数，下次可以直接运行 git push
git pull origin master从远程仓库更新内容到本地
git diff --staged查看暂存区的区别
git reset a.txt取消暂存a.txt文件

远程仓库

  • 显示远程仓库
    git remote列出每个远程库的简短名字。默认为origin
    git remote -v 此为 –verbose 的简写，取首字母，显示对应的克隆地址：
  • 添加远程仓库
    git remote add [shortname] [url]
  • 推送远程仓库
    git push -u origin master 将本地仓库推送到远程仓库（仓库名为origin 分支为master）-u 会保存当前推送的参数，下次可以直接运行 git push
    git pull origin master从远程仓库更新内容到本地
    推送前需要更新远程仓库

git diff 命令

git diff <filename> 查看文件在工作目录与暂存区的差别
git diff --cached <filename>查看已经 add 进暂存区但是尚未 commit 的内容差别

git 撤销

git checkout -- file 撤销工作区某一文件更改（此时还未暂存）
git reset --hard HEAD撤销工作区所有文件更改（此时还未暂存）
"git reset HEAD <file>取消暂存（此时暂存但未提交）
git revert HEAD 撤销提交一个版本(此时已提交（不管有没有推送），但是想撤销提交)，撤销完需要推送此次撤销
git revert commitID撤销到此次推送的ID版本（使用git log可查看推送ID）

分支

git branch dev 新建dev分支
git checkout dev 切换到dev分支
git checkout -b dev 新建并且切换到dev
git branch -d dev 删除dev分支
git branch 查看分支
git merge dev 合并dev分支到当前分支
git branch -D <name> 丢弃一个没有被合并过的分支,丢弃一个没有被合并过的分支
